Faz 1: Ön Gereksinimler ve Proje Başlatma

Bu ilk aşamada, geliştirme ortamımızı hazırlayacak ve temel proje iskeletini oluşturarak uygulamamıza ilk adımı atacağız. Gerekli araçların sisteminizde kurulu olduğundan emin olun.

  • Rust ve Cargo: Rust programlama dili ve paket yöneticisi.
  • Node.js ve npm/yarn/pnpm: JavaScript çalışma ortamı ve paket yöneticisi.

Faz 2: Temel Altyapının Kurulumu

Projemizin iskeletini oluşturduktan sonra, uygulamamızı daha profesyonel ve yönetilebilir hale getirecek kritik altyapı bileşenlerini ekleyeceğiz. Bu bölümde loglama, otomatik güncelleme ve frontend-backend iletişimi gibi konuları ele alacağız.

Loglama: `tauri-plugin-log` ile Gelişmiş Kayıt Tutma

Uygulamanızdaki olayları ve hataları izlemek için güçlü bir loglama altyapısı kurmak hayati önem taşır. `src-tauri/Cargo.toml` dosyanıza ilgili bağımlılığı ekleyin:

[dependencies]
tauri-plugin-log = { version = "1.3.0", features = ["colored"] }
log = "0.4.20"

Ardından `src-tauri/src/main.rs` dosyasında loglama eklentisini başlatın:

use tauri_plugin_log::{LogTarget, LoggerBuilder};

fn main() {
    tauri::Builder::default()
        .plugin(
            LoggerBuilder::new()
                .targets([
                    LogTarget::LogDir,
                    LogTarget::Stdout,
                    LogTarget::Webview,
                ])
                .build(),
        )
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}

Otomatik Güncelleme: Uygulamanızı Güncel Tutma

Tauri'nin yerleşik güncelleme mekanizması ile kullanıcılara yeni sürümleri kolayca dağıtabilirsiniz. İlk olarak, paketlerinizi imzalamak için bir anahtar çifti oluşturun:

npm run tauri signer generate -w ./src-tauri

Oluşturulan genel anahtarı (`pubkey`) `src-tauri/tauri.conf.json` dosyasındaki `updater` bölümüne ekleyin:

"updater": {
  "active": true,
  "endpoints": [
    "https://sizin-sunucunuz.com/updates.json"
  ],
  "dialog": true,
  "pubkey": "GENERATED_PUBLIC_KEY"
}

Güvenli İletişim (IPC): React ve Rust Arasında Köprü Kurma

Tauri, frontend (React) ve backend (Rust) arasında son derece güvenli bir iletişim kanalı sağlar. Bu, React arayüzünüzden Rust fonksiyonlarını çağırmanıza veya Rust tarafından arayüze anlık olaylar göndermenize olanak tanır.

IPC Akış Diyagramı

React Frontend
(Kullanıcı Arayüzü)
invoke ▶ (Komut Çağırma)
◀ emit (Olay Yayma)
Rust Backend
(Ana Mantık)

React'ten Rust'ı Çağırma: Rust tarafında `#[tauri::command]` ile bir fonksiyon tanımlayın ve React'ten `invoke` ile çağırın.

// Rust (main.rs)
#[tauri::command]
fn greet(name: &str) -> String {
    format!("Merhaba, {}!", name)
}

// React (App.tsx)
import { invoke } from '@tauri-apps/api/tauri';
const response: string = await invoke('greet', { name: 'React' });

Faz 3: JetBrains IDE'leri ile Hata Ayıklama

Verimli bir geliştirme süreci için hem frontend (React) hem de backend (Rust) kodunu rahatça ayıklayabilmek (debug) kritiktir. WebStorm ve RustRover için hata ayıklama konfigürasyonlarını aşağıda bulabilirsiniz.

WebStorm (Frontend)

"JavaScript Debug" konfigürasyonu oluşturun ve URL olarak Vite'ın çalıştığı adresi (genellikle `http://localhost:1420`) belirtin. Artık `.tsx` dosyalarına breakpoint koyabilirsiniz.

RustRover (Backend)

"Cargo Command" konfigürasyonu oluşturun. `Command` alanına `tauri`, `Additional arguments` alanına ise `dev` yazın. Bu sayede `.rs` dosyalarınıza breakpoint koyarak backend'i adım adım izleyebilirsiniz.
